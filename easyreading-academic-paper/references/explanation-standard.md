# 学术易读解释标准

目标是降低理解跳步，而不是降低技术内容。使用正式、可复核的语言，让读者同时知道“作者严格说了什么”“计算如何发生”“为什么这样设计”以及“证据支持到哪里”。

## 1. 证据标签

在容易混淆来源的位置使用以下标签：

- 论文内容：论文正文、公式、图表、caption 或 appendix 直接陈述。
- 官方代码：官方仓库中的实现、配置或注释。
- 外部背景：为理解论文补充的教材、综述或其他论文知识，给出来源。
- 分析者解释：不改变原结论的结构化解释、等价改写或中间推导。
- 根据上下文推断：原文未明确给出，但可从形状、代码或上下文推测；写明不确定性。

不要给每一句机械加标签。只在来源边界可能影响判断时使用。

## 2. 每个章节的解释顺序

按以下顺序展开原章节：

1. 本节任务：它要解决哪个子问题，依赖前文什么，产出供后文使用的什么。
2. 严格表述：按原文顺序完整转述定义、方法、论证、实验或讨论。
3. 必要前提：只补足理解这些内容必需的概念、符号和中间步骤。
4. 逐步展开：沿输入、变换、输出说明信息流或优化路径。
5. 作用与边界：说明该设计为何存在、依赖哪些假设、在哪些条件下可能失效。
6. 前后连接：把本节结果放回整篇论文的论证链。
7. 学习检查点：提出能由本节内容精确回答的技术问题。

理解桥梁必须紧邻它解释的内容，不要先堆完整章公式，再在数页之后统一解释。

## 3. 公式与定义

每个独立公式或公式组依次处理：

### 3.1 原式与来源

- 保留论文编号和原始数学关系。
- 改变符号、合并项或写成等价形式时，明确标注“等价改写”。
- 自己补出的中间式标注“分析者推导”，不能伪装成论文原式。

### 3.2 符号表

至少说明：

- 每个新符号的语义；
- 标量、向量、矩阵或张量形状；
- batch、时间、视角、像素、patch、点、Gaussian 或 token 维；
- 单位、取值范围、概率约束；
- 世界、相机、屏幕、局部或归一化坐标系；
- 训练变量、观测量、随机变量、条件变量和固定超参数。

原文没有给出形状或坐标系时写“论文未明确；根据上下文推断为……”，并说明推断依据。

### 3.3 计算顺序

沿真实执行顺序解释输入如何得到中间量和输出。说明广播、归一化、采样、排序、累积、mask、停止梯度或近似发生在哪里。

### 3.4 精确直觉

用一至数段说明为什么这种计算能完成目标。优先解释几何、概率、优化或信息流意义，不用生活化故事替代数学关系。

### 3.5 论文中的作用

指出公式属于：

- 表示或参数化；
- 前向模型、投影或渲染；
- 状态转移或生成过程；
- 训练目标、正则项或约束；
- 推理、采样或控制策略；
- 评估指标。

说明它与前后公式、模块和实验的关系。

### 3.6 梯度、假设与边界

需要时说明哪些量可微、梯度从哪里回传、近似如何影响优化，以及独立性、可见性、高斯性、马尔可夫性、静态场景等假设。

### 3.7 微型例子

只有在能消除具体歧义时加入数值或低维例子。明确标注这是分析者构造的例子，不得暗示为论文实验。

## 4. Figure 与架构图

每张图按以下顺序解释：

1. 给出原 Figure 编号和 caption 的核心含义。
2. 说明每个子图、坐标轴、颜色、线型、箭头、图例和输入输出。
3. 架构图按真实信息流逐步走读，标出训练专用与推理专用分支。
4. 结果图说明比较对象、控制变量、指标方向和不确定性。
5. 说明作者用这张图支持哪个结论，以及图本身不能证明什么。
6. 图像分辨率不足时标为 unavailable 或部分可读，禁止猜测数值。

可以用 Mermaid 重画信息流，但必须注明“结构化重绘”，且不能替代原图或删去原图中的分支。

## 5. Table 与实验数字

- 解释表头、行列分组、指标单位、上升/下降方向、加粗/下划线规则和缺失值。
- 覆盖所有 baseline 类别，不只讨论最佳方法。
- 同时给出绝对差值和必要的相对变化；不要把微小差距自动称为显著提升。
- 有均值、标准差、置信区间或多次运行时全部保留。
- 若论文未报告显著性检验，明确说明不能据此判断统计显著性。
- 区分“作者声称”“表格直接显示”和“分析者进一步解释”。

## 6. Algorithm、训练与推理流程

- 逐行解释输入、初始化、循环、条件、更新和返回值。
- 区分训练、验证和测试；区分离线预处理与在线推理。
- 说明停止条件、采样次数、迭代次数、随机种子和复杂度。
- 若正文算法与代码实现不同，分别陈述并给出证据，不擅自判定哪一个“才是真实方法”。

## 7. 实验章节的完整展开

按原文顺序覆盖：

1. 研究问题或假设；
2. 数据集、场景和划分；
3. 数据处理与增强；
4. baseline 与公平性条件；
5. 指标定义与方向；
6. 实现细节、训练资源和推理设置；
7. 主结果；
8. 消融与敏感性；
9. 效率、内存、速度和规模；
10. 定性结果；
11. 失败案例、负面结果和局限。

每项结果都回答“比较了什么、控制了什么、变化了多少、支持哪个结论、还不能说明什么”。不要用一张汇总表代替全部实验叙述。

## 8. Related Work 与 Discussion

- 保留作者采用的分类轴和技术谱系。
- 说明代表方法解决的问题、关键机制以及与本文的具体差异。
- 不将所有前作概括为“效率低、质量差、无法扩展”。
- Discussion 中区分作者自述限制与分析者判断。
- 不自动添加行业意义、科研选题或改进建议；只有用户明确要求时再扩展。

## 9. 可读性手段

允许并推荐：

- 首次出现的中英术语对照；
- 符号表、数据流表、坐标系表和模块对应表；
- 一段一论点，长句拆分但保留逻辑连接；
- 在每节开头说明任务，在结尾回扣论证链；
- 用 details 折叠冗长推导、补充实验和代码，同时完整保留内容；
- 对同一变量保持全篇一致命名。

避免：

- “你可以把它想象成……”式连续类比；
- 为追求轻松而删除条件、符号或反例；
- 用情绪化形容词代替数据；
- 先给夸张结论，再选择性寻找证据；
- “显而易见”“不难发现”等跳过推理的表达；
- “深入探讨”“具有重要意义”“带来新范式”等无具体信息的套话。
